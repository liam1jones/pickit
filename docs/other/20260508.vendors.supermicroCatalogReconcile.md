# PickIT ÔÇö Supermicro vendor map vs catalog shape

**Date (`date +%Y%m%d`):** `20260508` (from `date +%Y%m%d` on the authoring host)  
**Section:** Vendors  
**Subsection:** Supermicro catalog reconcile  

**Files**

- [`src/lib/cores/vendors/supermicro.ts`](../src/lib/cores/vendors/supermicro.ts) ÔÇö curated vendor overlay  
- [`src/lib/catalog/supermicro.ts`](../src/lib/catalog/supermicro.ts) ÔÇö auto-generated `CatalogItem[]` from NSIM Item Master (see `scripts/generateCatalog.mjs`)  
- [`src/lib/catalog/types.ts`](../src/lib/catalog/types.ts) ÔÇö `CatalogItem` interface  

---

## 1. Problem

`vendors/supermicro.ts` held a flat **`Record<partNumber, displayLabel>`** (string Ôćĺ `"Supermicro"`).  
`catalog/supermicro.ts` holds the **authoritative** part list as an **array of objects** matching `CatalogItem` (`partNumber`, `manufacturer`, `estimatedCost`, etc.). The two lived side by side with no structural link, so NSIM regen could drift from the hand-maintained key list without any type-level signal.

---

## 2. Target shape

Align the vendor module with the **same object fields** as catalog rows:

| Field | Role |
|--------|------|
| `id` | Stable slug from generator |
| `model` / `description` | NSIM long text |
| `manufacturer` | Legal OEM string (e.g. `Super Micro Computer Inc.`) |
| `category` | Coarse bucket (`hardware`, `transceiver`, ÔÇŽ) |
| `partNumber` | Join key between catalog and `VENDOR_MAP` |
| `estimatedCost` / `specs` | Catalog presentation |
| `vendorIds` | `['supermicro']` in generated file |

The global merge in [`src/lib/cores/vendors/index.ts`](../src/lib/cores/vendors/index.ts) still needs **`Record<string, string>`** for PickIT UI (`VENDOR_MAP[pn]`). That record is now **derived** from the filtered catalog array so there is a single row-level source of truth.

---

## 3. Restructuring approach

1. **Keep a curated part-number set** (`SUPERMAP_PART_NUMBERS`) ÔÇö same 17 SKUs that previously appeared in the inline `VENDOR_MAP`. The full Supermicro catalog is much larger; not every catalog row should inject a short vendor label into the global map (other parts already carry manufacturer via catalog in PartSearch).

2. **Filter** `supermicroCatalog` (default export from `catalog/supermicro.ts`) with `Set.has(partNumber)` so each surviving row is a full **`CatalogItem`**, not a hand-duplicated stub.

3. **Sort** the resulting array by `partNumber` with numeric-aware `localeCompare`. Catalog generator order can change between CSV runs; a stable sort keeps git diffs readable and matches how we reason about ÔÇťlist of partsÔÇŁ in reviews.

4. **Build** `supermicroVendorPartMap` with `Object.fromEntries(rows.map(...))` so the merged `VENDOR_MAP` behavior stays **`"Supermicro"`** in the UI (short label), while `manufacturer` on each row remains the long legal name from NSIM ÔÇö intentional distinction documented in code.

5. **Dev-only guard** ÔÇö if a curated PN disappears from the catalog after regen, `supermicroVendorCatalogItems.length` will not match the set size; we `console.warn` in `import.meta.env.DEV` so local runs surface the mismatch without breaking production builds.

---

## 4. What we did not do

- **Did not** replace the entire historical `VENDOR_MAP` Supermicro section with ÔÇťall rows in `supermicro.ts`ÔÇŁ ÔÇö that would flood unrelated PNs into the overlay and duplicate catalog metadata paths.  
- **Did not** change `CatalogItem` or the generator ÔÇö reconciliation is at the **vendor overlay** layer only.  

---

## 5. Follow-ups (optional)

- Apply the same pattern to other vendor files under `src/lib/cores/vendors/` where the part set is a subset of a single `catalog/<slug>.ts` file.  
- If a part moves to another manufacturer in NSIM, update `SUPERMAP_PART_NUMBERS` or drop the override and rely on catalog `manufacturer` only.  

---

## 6. Verification

- `npm run typecheck` ÔÇö `supermicro.ts` imports `CatalogItem` and the catalog default export.  
- `npm run build` ÔÇö Vite bundles `catalog/supermicro` through the vendor module.  
- In dev, if counts mismatch after `node scripts/generateCatalog.mjs`, the console warning should fire until the curated set is fixed.
